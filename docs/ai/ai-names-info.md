# AI名词汇总
> 持续更新
##### 模型

基于数学算法（如神经网络、决策树等）训练得到的函数，用于从输入数据中学习规律，并对新的数据进行预测或生成结果。

##### 语言模型

语言模型是一种用于建模自然语言概率分布的模型，核心任务是根据上下文预测下一个词的概率。现代语言模型通常基于 Transformer 架构，通过大规模预训练学习语言结构和语义关系。

##### 大语言模型-LLM

语言模型随着权重文件越来越大，在某一个临界点出现了智能，这就是大语言模型。此时LLM只能做文字接龙，也就是预测下一个字是什么。

##### Prompt

用户输入给模型的文本。

##### Context

模型在一次推理中能够看到的完整上下文，包括**系统提示**、**历史对话**和**当前问题**。

##### Memory

指系统在对话外保存的长期信息，这些信息会被拼接进 Prompt 形成 Context，本质上模型本身并**不具备持久记忆能力**。

##### agent

Agent 是在 LLM 之上构建的一种智能执行机制，它能够根据任务需求自主决定调用外部工具、API 或程序，以获取模型本身无法直接访问的信息或完成实际操作。

##### RAG

模型回答前先去数据库检索相关资料，再结合资料生成答案。

##### web-search

模型联网搜索用的工具。

##### Function Calling

Function Calling 是一种让 LLM 以结构化方式调用外部函数或 API 的机制。模型不会直接执行函数，而是生成函数调用参数，由外部系统负责实际执行。

##### MCP

是一种用于标准化大语言模型与外部工具交互的通信协议，它定义了模型如何声明、调用和接收工具执行结果，从而实现模型能力的可扩展化。

##### subAgent

对于一些复杂任务，部分独立子任务可以交给其他agent去完成。（实际上是上下文隔离）

##### LangChain 

是一个用于构建大语言模型应用的开发框架，提供了模型调用、工具接入、记忆管理和流程编排等能力。

##### Workflow 

是一种任务流程设计模式，用于组织多个步骤的执行逻辑，一般指的是图形化的拖拽。

##### skill

就是一个懒加载的prompt加载器，把一些复杂流程由langchain写死变为描述成流程放在md里面，从而更加灵活。感觉可以取代mcp。



##### mcp与Function Calling区别

Function Calling 是模型调用（自定义）函数的能力；MCP 是让模型以标准方式连接外部世界的协议。

非常重要的一点：

> MCP 可以基于 Function Calling 实现

也就是说：

- 有些模型通过 function calling 实现 MCP client
- 但 MCP 不等于 function calling
- MCP 是更高层协议
