# openclaw使用体验

想着毕设本来就要部署自己的项目，再加上也想亲自体验一下 OpenClaw 的具体功能，于是咬咬牙（其实也没太心疼）入手了一台腾讯云 2C4G 的服务器。

原本打算自己手动部署 OpenClaw 的，结果发现腾讯云已经把它做成了应用镜像的形式，一键安装就能用，直接省去了在命令行里折腾配置和环境依赖的过程。

开箱即用这块，腾讯云做的都挺好的，就是这个skills好像有点bug，不管安装多少都只显示默认的那个skill。所以只能去服务器自己手动安装（或者直接叫openclaw去安装，他有你服务器的权限）。

下面简单聊聊我的使用体验。
当 Agent 拥有服务器的完全权限时，能力确实非常强大，很多操作几乎可以一步到位；但与此同时，这种“全权限”也意味着更高的安全风险，一旦出现误操作或被恶意利用，后果可能会比较严重。

比如我一直想做的一件事：每天早上都能自动收到一份全球范围内最新的后端技术动态汇总，涵盖高并发、数据库、分布式等方向。去 ClawHub 看了一下，发现这个需求其实并不复杂，通过组合几个现成的 skills，就可以很轻松地实现自动抓取、整理和推送的完整流程。效果如下：

经过检验，这些文章都是真实的，而且确实是符合我要求的。要是以前的话，我只能手动去写定时任务，去写爬虫脚本去爬取各大社区的新内容，openclaw帮我轻松完成了这个任务，用时几分钟，花费0.3元。

但是因为核心工具 `blogwatcher` 是一个用 Go 语言编写的 CLI 工具，需要通过 Go 来从源码安装，所以他给我下载了一个go，在完全没经过我同意的情况下，无论是版本还是什么，都没和我沟通过。如果这个skills被下毒了，后果简直不堪设想，还好我看openclaw官方已经和专门的威胁检测平台达成了合作，这方面的担忧也可以先放放了。

ai发展太快了，新的环境应该不会从双休变成三休，只会是降本增效居多。都说初级和中级将被淘汰，但是没初级和中级哪来的高级，还是好好巩固基础，多学多做吧，什么都依靠ai只会害了我们，保持持续的高能量核心驱动力才是职业顺利的万金油。
