# 死锁梳理

### 一、什么是死锁

如果一组进程中的**每个进程**都在等待数组中其他进程占有资源，那么这组进程就处于死锁状态。

比如下面的例子：

```text
线程A：
  lock(resource1)
  lock(resource2)

线程B：
  lock(resource2)
  lock(resource1)
```

---



### 二、死锁的4个必要条件

死锁的产生必须同时满足以下的4个条件：

- 互斥条件：资源一次只能被一个进程所使用。
  - 比如锁数据库连接等，也就是资源如果允许共享，就不会发生死锁。
- 请求与保持：完成已经持有资源，同时继续申请新的资源。
  - 也就是如果进程必须一次申请全部资源，就不会发生这个条件。
- 不可剥夺：资源不能被强占，只有持有资源的进程**主动释放**。
- 循环等待：存在进程等待环。

### 四、死锁处理对策
#### 1. 死锁预防

核心思想是破坏四个条件其中的一个。

互斥 - 资源共享
保持等待 - 一次申请全部资源
不可剥夺 - 强制释放资源
循环等待 - 资源按顺序申请

#### 2. 死锁避免

**银行家算法**：特点是每次分配资源之前，检测系统是否安全。

一般在操作系统使用，实际的系统较少使用。

#### 3. 死锁检测

允许死锁发生，系统 **定期检测是否有死锁**。

方法：检测资源分配图是否有环
